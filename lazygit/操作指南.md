## 1. 简介

Lazygit 是一个简单的终端 UI 工具，为 git 命令提供可视化界面。它将常用的 git 操作封装为按键快捷方式，让你无需记忆复杂的命令行参数即可高效管理 git 仓库。

**核心优势：**
- 可视化暂存单个代码块/行
- 交互式 rebase 变得极其简单
- Cherry-pick 通过复制粘贴完成
- 一键撤销/重做 git 操作
- 支持自定义命令和补丁操作

## 2. 安装

```bash
# macOS (Homebrew)
brew install lazygit

# Ubuntu/Debian 13+
sudo apt install lazygit

# Arch Linux
sudo pacman -S lazygit

# Fedora
sudo dnf copr enable dejan/lazygit && sudo dnf install lazygit

# Go 安装
go install github.com/jesseduffield/lazygit@latest

# 验证安装
lazygit --version
```

更多安装方式见 [README.md](README.md)。

## 3. 界面概览

Lazygit 启动后分为以下几个面板：

```
┌──────────────┬─────────────────────────────────┐
│  Status      │                                 │
├──────────────┤                                 │
│  Files       │        主视图                    │
├──────────────┤     (Commit / Diff)             │
│  Branches    │                                 │
├──────────────┤                                 │
│  Commits     │                                 │
├──────────────┼─────────────────────────────────┤
│              命令日志 (Command Log)              │
└───────────────────────────────────────────────┘
```

- **左侧面板**：Status、Files、Branches、Commits 等标签页
- **右侧主视图**：显示选中项的详细信息（diff、commit 内容等）
- **底部命令日志**：显示已执行的 git 命令及其输出

**面板切换：**
- `1` ~ `5`：快速跳转到对应面板
- `Tab`：在当前面板的标签页间切换
- `0`：聚焦主视图
- `+` / `_`：切换屏幕模式（普通/半屏/全屏）

## 13. 合并与变基

### 13.1 合并冲突解决

当出现合并冲突时，在文件面板按 `M` 查看合并选项。进入冲突解决视图后：

| 按键 | 操作 | 说明 |
|------|------|------|
| `<space>` | 选择区块 | 接受当前区块 |
| `b` | 选择所有区块 | 接受所有区块 |
| `<up>` / `<down>` | 选择顶部/底部块 | 切换冲突的版本 |
| `<left>` / `<right>` | 上一个/下一个冲突 | 跳转到其他冲突 |
| `z` | 撤销 | 撤销上次冲突解决 |
| `e` | 编辑文件 | 用外部编辑器解决冲突 |
| `M` | 合并冲突选项 | 查看更多选项 |
| `<esc>` | 返回 | 返回文件面板 |

### 13.2 交互式变基

1. 在 Commits 面板选择一个提交，按 `i` 开始交互式变基
2. 对 TODO 列表中的提交进行操作：
   - `s`：压缩到下方提交
   - `f`：修正（fixup）到下方提交
   - `d`：删除提交
   - `e`：编辑提交
   - `<ctrl+j>` / `<ctrl+k>`：上下移动提交
3. 按 `m` 打开变基选项菜单，选择 continue/abort/skip

### 13.3 标记基准变基

适用于从非主分支分出的特性分支：

1. 找到特性分支的基准提交
2. 按 `B` 标记为基准提交（会以黄色高亮）
3. 在目标分支上按 `r` 进行变基
4. 只有基准提交之后的提交会被变基过去

## 14. 撤销与重做

| 按键 | 操作 |
|------|------|
| `z` | 撤销 |
| `Z` | 重做 |

**工作原理：**
- 基于 reflog 实现，逐条回溯 git 操作历史
- 支持撤销在 lazygit 外执行的 git 命令
- 撤销操作本身也会记录在 reflog 中

**限制：**
- 只能撤销记录在 reflog 中的操作（提交、分支切换等）
- 无法撤销对工作树的直接修改或 stash 操作
- 推送到远程的操作不可撤销
- 变基进行中不支持撤销（请使用 `m` 中止变基）

## 16. 自定义补丁（Custom Patches）

Lazygit 支持从提交中构建自定义补丁进行精细操作。

### 16.1 构建补丁

1. 在 Commits 面板按 `<enter>` 查看提交的文件
2. 选择文件按 `<enter>` 进入逐行视图
3. 用 `<space>` 标记/取消行加入补丁
4. 用 `a` 切换代码块选择模式
5. 用 `v` 启用拖动选择

### 16.2 补丁操作

构建好补丁后，按 `<ctrl+p>` 打开补丁选项菜单：

- **从当前提交移除补丁**：将选中的行从该提交中删除
- **从补丁创建新提交**：将选中的行拆分为新提交
- **反向应用到索引**：将补丁反向应用到暂存区
- **复制到新分支**：将补丁应用到新分支

### 16.3 从提交中移除行

在提交文件视图中，按 `<enter>` 进入文件，选择要移除的行，按 `d` 直接从提交中移除选中的行（后台执行交互式变基）。

---

## 17. Fixup 工作流

### 17.1 场景

PR 审查中需要修改之前提交的代码，但又希望审查者能看到你改了什么。

### 17.2 操作步骤

**创建 fixup 提交：**
1. 在 Commits 面板找到需要修改的提交
2. 按 `F` 创建 fixup 提交（自动以 `fixup!` 为前缀）

**创建 amend 提交（可同时修改提交信息）：**
1. 按 `F` 后选择 amend 选项
2. 输入新的提交信息

**自动查找基准提交：**
1. 在 Files 面板完成变更
2. 按 `<ctrl+f>` 自动找到变更对应的原始提交
3. 按 `F` 创建 fixup 提交

**自动压缩 fixup 提交：**
1. 在 Commits 面板选择分支的第一个提交
2. 按 `S` 自动压缩该提交之上的所有 `fixup!` 提交

---

## 18. 堆叠分支（Stacked Branches）

当大型特性需要拆分为多个小分支时，可以使用堆叠分支：

1. 设置 git 配置：`git config rebase.updateRefs true`
2. 创建多个分支，后一个基于前一个
3. 堆叠的分支头部在 lazygit 中以**青色星号**标记
4. 变基最顶层分支时，其他堆叠分支会自动跟随

## 19. 自定义命令

在 `config.yml` 中配置自定义命令：

```yaml
customCommands:
  - key: 'X'
    description: "自定义操作菜单"
    commandMenu:
      - key: 'c'
        command: 'git format-patch --stdout {{.SelectedCommitRange.From}}^..{{.SelectedCommitRange.To}} | pbcopy'
        context: commits
        description: "复制选中的提交为补丁"
```

**上下文类型：**
`status`、`files`、`localBranches`、`remoteBranches`、`remotes`、`tags`、`commits`、`reflogCommits`、`stash`、`submodules`、`worktrees`、`global`

**支持的提示类型：**
- `input`：文本输入
- `confirm`：确认对话框
- `menu`：选项菜单
- `menuFromCommand`：从命令输出生成菜单

## 21. 常用工作流示例

### 21.1 快速提交并推送

```
1. 在 Files 面板按 space 暂存文件
2. 按 c 提交，输入消息，按 enter 确认
3. 按 P 推送
```

### 21.2 修复 PR 审查意见

```
1. 修改代码
2. 在 Files 面板暂存相关变更
3. 按 <ctrl+f> 自动找到原始提交
4. 按 F 创建 fixup 提交
5. 按 S 自动压缩所有 fixup 提交
6. 强制推送
```

### 21.3 Cherry-pick 提交

```
1. 在源分支的 Commits 面板选择提交
2. 按 C 复制（可选择多个）
3. 切换到目标分支
4. 在 Commits 面板按 V 粘贴
```

### 21.4 整理提交历史

```
1. 在 Commits 面板按 i 开始交互式变基
2. 选择要整理的提交范围
3. 对每个提交：s(压缩)/f(修正)/d(删除)/e(编辑)
4. 用 <ctrl+j>/<ctrl+k> 调整顺序
5. 按 m 打开菜单，选择 continue
```

### 21.5 比较两个提交/分支

```
1. 在第一个提交/分支上按 W 打开 diff 菜单
2. 选择 "与选中项比较"
3. 选择第二个提交/分支
4. 主视图显示差异，按 enter 查看文件详情
5. 按 W 可反转比较方向或退出比较模式
```

### 21.6 堆叠分支工作流

```
1. git config rebase.updateRefs true
2. 在 main 分支上创建 feature-base 分支
3. 在 feature-base 上创建 feature-frontend 分支
4. 在 feature-frontend 上创建 feature-backend 分支
5. 变基最顶层分支时，底层分支自动跟随
```
