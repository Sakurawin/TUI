# nnn 中文操作指南

> `nnn`（n³）是一个极简、快速、几乎零配置的终端文件管理器。它的设计目标不是“像图形文件管理器那样什么都做”，而是让你用最少的按键完成浏览、筛选、选择、复制、移动、重命名、预览和启动外部工具等日常文件工作。

## 1. nnn 适合什么场景

`nnn` 适合已经习惯终端工作流的人，尤其是下面这些场景：

- 你经常在终端里浏览大量文件，不想频繁切换到图形界面。
- 你需要快速筛选某个目录中的文件，但又不想手敲长命令。
- 你希望用一套统一的键盘操作完成复制、移动、删除、重命名、批处理。
- 你经常在多个项目目录之间来回切换，需要“标签页”式上下文。
- 你想把文件管理和外部工具结合起来，比如预览、压缩、挂载、查找、对比、上传。
- 你在纯 CLI 环境、远程 SSH、WSL、Termux、服务器或资源受限设备上工作。

如果你的工作流是“打开目录、找文件、选文件、处理文件、退出”，`nnn` 很合适。
如果你需要大量鼠标操作、复杂树状面板、重度拖拽交互，它不是最佳选择。

## 2. 核心特点

根据项目 README，`nnn` 的核心定位可以概括为：

- 小：二进制体积很小，内存占用低。
- 快：强调少读写、少刷新、搜索和跳转响应快。
- 简：默认几乎不需要配置。
- 强：支持上下文、选择、过滤、插件、会话、书签、预览、回收站、挂载等。
- 可扩展：核心保持轻量，很多“附加能力”通过插件实现。

## 3. 安装与启动

### 3.1 安装

不同发行版可以直接用包管理器安装。源码编译也很直接：

```bash
git clone https://github.com/jarun/nnn.git
cd nnn
make
sudo make install
```

如果你只想快速体验，安装后直接运行 `nnn` 即可。

### 3.2 基本启动方式

```bash
nnn
nnn /path/to/dir
nnn /path/to/file.txt
nnn -d
nnn -H
nnn -e
```

常见含义：

- `nnn`：打开当前目录。
- `nnn /path/to/dir`：直接进入目标目录。
- `nnn /path/to/file.txt`：自动定位到文件所在目录。
- `-d`：详细模式，显示更多文件信息。
- `-H`：显示隐藏文件。
- `-e`：使用编辑器打开文本文件。

### 3.3 推荐的初始启动思路

第一次使用时，先记住这几个动作就够了：

- `↑/↓` 或 `j/k`：移动。
- `Enter`：进入目录或打开文件。
- `h` 或 `←`：返回上级目录。
- `/`：过滤。
- `Space`：选择文件。
- `q`：退出当前上下文。
- `?`：查看帮助和快捷键。

## 4. 基本交互模型

`nnn` 的工作方式可以理解为：

1. 光标停在某个条目上。
2. 用快捷键直接对当前条目执行动作。
3. 需要批量处理时，把条目放入“选择缓冲区”。
4. 切换上下文、书签、会话或插件继续工作。

它不是“层层菜单型”的文件管理器，而是“按键即动作”的工具。

## 5. 常用快捷键

### 5.1 基本导航

| 快捷键 | 功能 |
|---|---|
| `↑/k` | 上移 |
| `↓/j` | 下移 |
| `←/h` | 返回上级目录 |
| `→/l` | 进入目录或打开文件 |
| `Enter` | 打开/进入 |
| `q` | 退出当前上下文 |
| `Ctrl+Q` | 完全退出 |

### 5.2 快速跳转

| 快捷键 | 功能 |
|---|---|
| `g` / `Home` | 跳转到第一个条目 |
| `G` / `End` | 跳转到最后一个条目 |
| `~` | 跳转到 HOME 目录 |
| `@` | 跳转到启动目录 |
| `-` | 跳转到上次访问目录 |
| `` ` `` | 跳转到根目录 `/` |
| `'` | 跳转到第一个非目录文件 |

### 5.3 帮助与信息

| 快捷键 | 功能 |
|---|---|
| `?` | 显示帮助和配置屏幕 |
| `f` / `Ctrl+F` | 显示文件详细信息 |

## 6. 文件浏览与导航

### 6.1 浏览目录

- 用 `j/k` 或方向键移动光标。
- 用 `Enter`、`l` 或 `→` 进入目录。
- 用 `h` 或 `←` 返回上级目录。
- 用 `Backspace` 也可以返回上级目录。

### 6.2 显示隐藏文件

- `.`：切换显示隐藏文件。
- `-H`：启动时显示隐藏文件。

### 6.3 详细模式

- `d`：切换详细模式。

详细模式更适合查看权限、大小、时间等信息，适合做文件筛选和排查。

### 6.4 Type-to-nav

`nnn` 支持 type-to-nav，也就是输入字符后快速定位到匹配内容。

```bash
nnn -n
```

使用场景：你已经知道目标目录或文件名的大致开头，不想先进入过滤模式。

## 7. 文件操作

### 7.1 新建文件或目录

- `n`：新建文件或目录。

输入规则：

- 文件名直接输入，如 `notes.txt`
- 目录名以 `/` 结尾，如 `project/`
- 支持多级目录，如 `src/components/`

### 7.2 重命名

- `Ctrl+R`：重命名当前文件。
- `r`：批量重命名，通常会进入编辑器/批处理流程。

### 7.3 编辑

- `e`：用 `$EDITOR` 或 `$VISUAL` 打开文件。

### 7.4 复制、移动、删除

| 快捷键 | 功能 |
|---|---|
| `p` / `Ctrl+P` | 复制选择项到当前位置 |
| `v` / `Ctrl+V` | 移动选择项到当前位置 |
| `w` / `Ctrl+W` | 复制/移动并重命名 |
| `x` / `Ctrl+X` | 删除到回收站 |
| `X` | 强制删除 |

### 7.5 权限与压缩

| 快捷键 | 功能 |
|---|---|
| `*` | 切换可执行权限 |
| `z` | 创建压缩包 |

## 8. 选择与批量处理

这是 `nnn` 最重要的能力之一。

### 8.1 如何选择

| 快捷键 | 功能 |
|---|---|
| `Space` / `+` | 切换当前文件选择状态 |
| `m` | 进入多选模式 |
| `a` | 选择当前目录所有文件 |
| `A` | 反选当前目录 |
| `E` | 编辑选择列表 |

### 8.2 选择后的常用动作

选择完之后，可以把选择集作为一个整体处理：

- `p`：复制到当前目录。
- `v`：移动到当前目录。
- `x`：删除选中项。
- `z`：打包。
- `w`：复制或移动并重命名。

### 8.3 选择适合什么

- 批量拷贝照片、文档、下载文件。
- 跨目录整理资源。
- 一次性删除一批临时文件。
- 把多个文件打包发给别人。

### 8.4 选择的特点

- 选择可以跨目录保留。
- 选择可在多个 `nnn` 实例间共享。
- 选择状态通常保存在 `~/.config/nnn/.selection`，也可以通过环境变量自定义路径。

## 9. 过滤、搜索与排序

### 9.1 过滤

- `/`：进入过滤模式。
- 输入字符后，文件列表会实时缩小。
- `Esc`：退出过滤。
- `Ctrl+L`：清除过滤或应用上次过滤，取决于当前状态。

### 9.2 过滤模式

`nnn` 支持多种过滤风格：

- 字符串过滤：默认模式。
- 模糊过滤：`-z`。
- 正则过滤：`-g`。

切换模式时，通常在空过滤提示符下操作。

### 9.3 自动进入目录

当过滤结果唯一且它是目录时，`nnn` 会自动进入该目录。

如果你不想要这个行为，可以用 `-A` 关闭。

### 9.4 排序

| 快捷键 | 功能 |
|---|---|
| `t` | 打开排序菜单 |
| `Ctrl+T` | 在常见排序方式间循环 |

常用排序方式包括：

- 名称
- 大小
- 时间
- 扩展名
- 访问/修改/变更时间
- 数值自然排序

如果目录结构复杂，按时间或大小排序通常最实用；如果你在处理大量按编号命名的文件，自然排序更方便。

## 10. 上下文、书签与会话

### 10.1 上下文

`nnn` 支持多个上下文，类似标签页或工作区。

| 快捷键 | 功能 |
|---|---|
| `1-8` | 切换到指定上下文 |
| `Tab` | 顺序切换上下文 |
| `Shift+Tab` | 反向切换上下文 |

上下文适合同时处理多个项目目录，比如“源码、文档、构建产物、下载目录”并行工作。

### 10.2 书签

书签有两种常见形式：

- 通过 `NNN_BMS` 定义快捷键书签。
- 通过目录快捷方式或符号链接维护固定路径。

示例：

```bash
export NNN_BMS="d:$HOME/Docs;u:/home/user/Cam Uploads;D:$HOME/Downloads/"
```

常用操作：

| 快捷键 | 功能 |
|---|---|
| `b` | 打开书签列表 |
| `B` | 创建当前目录书签 |

### 10.3 会话

会话用于保存浏览状态，适合长期项目。

| 快捷键 | 功能 |
|---|---|
| `s` | 管理会话 |

常见模式：

```bash
nnn -S
nnn -s mysession
```

会话通常会保存：

- 各上下文目录状态
- 排序状态
- 过滤状态
- 当前工作目录

### 10.4 `cd on quit`

如果你希望退出 `nnn` 后自动回到当前目录，可以在 shell 中包装一个函数。项目 README 也建议这样做。

思路是：

1. `nnn` 退出时把路径写入临时文件。
2. shell 读取这个临时文件。
3. 当前 shell 的 `PWD` 随之切换。

这对日常终端工作流非常有用。

## 11. 插件系统

插件是 `nnn` 扩展能力的核心。

### 11.1 插件是什么

插件本质上是可执行脚本或二进制。`nnn` 负责把当前文件、目录和环境信息传给插件，插件完成更复杂的动作。

### 11.2 安装插件

官方提供了批量安装脚本：

```bash
sh -c "$(curl -Ls https://raw.githubusercontent.com/jarun/nnn/master/plugins/getplugs)"
```

默认安装目录一般是：

```bash
${XDG_CONFIG_HOME:-$HOME/.config}/nnn/plugins
```

### 11.3 配置插件快捷键

```bash
export NNN_PLUG='f:finder;o:fzopen;p:mocq;d:diffs;t:nmount;v:imgview'
```

使用方式：

- `;`：打开插件提示符。
- `;f`：运行 `finder`。
- `;o`：运行 `fzopen`。
- `Alt+键`：也可以作为替代触发方式。

### 11.4 运行命令作为插件

`NNN_PLUG` 不一定只能绑定脚本，也可以直接绑定命令：

```bash
export NNN_PLUG='x:!chmod +x "$nnn";g:!git log'
```

规则要点：

- `!` 表示执行命令。
- `*` 表示命令执行后跳过确认提示。
- `&` 表示以 GUI 方式运行。
- `|` 表示分页显示命令输出。
- `>` 表示在浮动窗口中显示输出。

### 11.5 常见插件用途

| 插件 | 用途 |
|---|---|
| `preview-tui` | 文件预览 |
| `nuke` | CLI 环境下打开文件 |
| `find` 类插件 | 目录内查找 |
| `diffs` | 文件或目录对比 |
| `nmount` | 挂载/卸载设备 |
| `renamer` | 批量重命名 |
| `organize` | 按类型整理文件 |
| `imgview` | 图片查看和操作 |

### 11.6 插件能拿到什么信息

插件运行时通常会收到：

- 当前悬浮文件名
- 当前工作目录
- 选择模式输出文件
- `NNN_PIPE`
- `NNN_FIFO`
- 隐藏文件状态
- 选择偏好状态

这意味着插件不仅能“读文件”，还可以向 `nnn` 回写状态，实现联动。

## 12. 常用配置

### 12.1 常见环境变量

```bash
export NNN_OPTS="cEnrx"
export NNN_OPENER=nuke
export NNN_BMS="d:$HOME/Docs;D:$HOME/Downloads"
export NNN_PLUG='f:finder;o:fzopen;p:preview-tui'
export NNN_COLORS='12341234'
export NNN_FCOLORS='c1e2272e006033f7c6d6abc4'
export NNN_ORDER='t:/home/user/Downloads;S:/tmp'
export NNN_ARCHIVE="\\.(7z|bz2|gz|tar|tgz|zip)$"
export NNN_SSHFS='sshfs -o reconnect,idmap=user'
export NNN_RCLONE='rclone mount --read-only --no-checksum'
export NNN_TRASH=1
export NNN_SEL='/tmp/.sel'
export NNN_FIFO='/tmp/nnn.fifo'
export NNN_TMPFILE='/tmp/.lastd'
```

### 12.2 这些变量分别做什么

- `NNN_OPTS`：设置启动选项组合。
- `NNN_OPENER`：设置默认打开器。
- `NNN_BMS`：设置书签。
- `NNN_PLUG`：设置插件键位。
- `NNN_COLORS`：设置上下文颜色。
- `NNN_FCOLORS`：设置文件类型颜色。
- `NNN_ORDER`：设置特定目录的排序规则。
- `NNN_ARCHIVE`：设置压缩包识别规则。
- `NNN_SSHFS` / `NNN_RCLONE`：设置远程挂载命令。
- `NNN_TRASH`：启用回收站删除。
- `NNN_SEL`：自定义选择文件路径。
- `NNN_FIFO`：用于预览联动和数据流。
- `NNN_TMPFILE`：用于退出时保存目录。

### 12.3 CLI-only 环境建议

如果你在纯终端环境使用 `nnn`，README 建议把 `NNN_OPENER` 指向 `nuke`，并配合 `-c` 使用。这样可以在没有桌面环境时仍然正常打开文件。

### 12.4 推荐的 shell 包装函数

如果你经常想让 `nnn` 退出后自动 `cd`，建议在 shell 里包一层函数，而不是每次手动进入目录。

## 13. 应用场景示例

### 场景 1：快速浏览下载目录

适合在 `Downloads`、临时目录、构建产物目录中快速找文件。

```bash
nnn ~/Downloads
/
pdf
```

做法：进入目录后直接过滤，找到目标后回车打开或进一步操作。

### 场景 2：批量整理照片

适合把相机导入文件按日期、项目或内容分类。

```bash
Space
Space
p
```

做法：先选择一批照片，再复制到目标目录。也可以配合插件做批量重命名、缩略图预览。

### 场景 3：在两个项目目录之间搬文件

适合源代码、资源文件、配置模板之间的迁移。

```bash
1
# 在上下文 1 选择源文件
2
# 到上下文 2 打开目标目录
v
```

做法：把源目录和目标目录放在两个上下文里，减少来回切换成本。

### 场景 4：项目初始化

适合快速创建目录结构。

```bash
n
myproject/
n
src/
n
docs/
```

做法：通过 `n` 快速创建树状目录，然后逐层进入补文件。

### 场景 5：预览文本、图片和二进制相关信息

适合在不离开终端的情况下查看内容。

```bash
export NNN_PLUG='p:preview-tui'
export NNN_FIFO='/tmp/nnn.fifo'
nnn
;p
```

做法：用预览插件把“选中文件是什么”这件事变成即时反馈。

### 场景 6：做文件对比

适合检查两个版本、两个目录、两个配置文件的差异。

```bash
export NNN_PLUG='d:diffs'
```

做法：把待比较文件选中后交给插件。

### 场景 7：批量重命名

适合整理照片、录音、下载文件、导出文件。

```bash
export NNN_PLUG='r:renamer'
```

做法：先选中文件，再进入批量重命名插件或内置批处理流程。

### 场景 8：远程挂载和手机文件交换

适合通过 `sshfs`、`rclone`、`MTP`、`kdeconnect` 在本地终端里管理远程内容。

做法：把远程资源挂载成目录，然后用 `nnn` 当成本地文件管理器处理。

### 场景 9：纯 CLI 环境打开文件

适合服务器、SSH、Termux、无桌面环境。

```bash
export NNN_OPENER=nuke
nnn -c
```

做法：通过 `nuke` 或自定义 opener 打开文本、媒体和脚本文件。

### 场景 10：日常“终端即工作台”

适合把 `nnn` 当成 shell 流程的一部分，而不是独立工具。

做法：

- 用书签快速跳目录。
- 用上下文分开项目。
- 用插件做预览、查找、对比、挂载。
- 用 `cd on quit` 把当前目录带回 shell。

## 14. 常见问题

### 14.1 文件颜色不显示

- 检查终端是否支持颜色。
- 先确认 `TERM` 是否正常。

### 14.2 中文文件名乱码

- 检查 `LANG` 和 `LC_ALL` 是否使用 UTF-8。

### 14.3 插件不工作

- 检查插件是否有执行权限。
- 检查 `NNN_PLUG` 是否配置正确。
- 检查插件文件是否放在正确目录。

### 14.4 预览不显示

- 确认预览插件已安装。
- 确认 `NNN_FIFO` 已设置。
- 确认对应预览器或依赖程序存在。

### 14.5 退出后没有自动回到目录

- 检查 shell 包装函数是否生效。
- 检查 `NNN_TMPFILE` 是否指向正确路径。

## 15. 最后建议

`nnn` 最有效的使用方式不是“记住所有按键”，而是先掌握以下 8 个动作：

- 浏览：`j/k`、`Enter`、`h`
- 过滤：`/`
- 选择：`Space`
- 复制/移动：`p` / `v`
- 删除：`x`
- 上下文：`Tab`
