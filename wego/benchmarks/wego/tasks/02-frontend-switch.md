# 任务 02：前端切换

## 指令

使用多个前端运行 `wego`，并验证每个前端都会产生预期的输出格式。

## 前置条件

- 使用本地 JSON 固定文件以避免网络波动带来的差异。
- 在仅切换前端时，保持 backend、location 和 days 不变。

## 收集的证据

- `json` 的标准输出
- `markdown` 的标准输出
- `ascii-art-table` 的标准输出
- 标准错误 (stderr)
- 退出码

## 判定依据

- 类型：标准输出判定 + 退出状态判定
- 成功条件：
  - `json` 前端输出可解析为有效 JSON
  - `markdown` 前端输出包含稳定的 markdown 表格结构
  - `ascii-art-table` 输出在视觉上与前两种格式明显不同
  - 所有运行都成功退出
- 失败条件：
  - 切换前端没有改变输出结构
  - `json` 输出无法解析
  - 有效前端执行时发生错误退出

## 负面案例

- 使用无效的前端名称，并确认判定机制记录为失败。

## 边界情况

- 如果结构特征仍然正确，允许存在空格或 ANSI 颜色序列等格式差异。
