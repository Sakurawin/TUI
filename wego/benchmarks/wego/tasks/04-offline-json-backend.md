# 任务 04：离线 JSON 后端执行

## 指令

使用本地 JSON 固定文件运行 `wego`，并验证它可以在不依赖外部网络访问的情况下成功执行。

## 前置条件

- 固定文件 `benchmarks/wego/fixtures/weather-sample.json` 存在。
- 使用 `--backend json`。
- 为了稳定解析，优先使用 `--frontend json` 或 `--frontend markdown`。

## 收集的证据

- 标准输出
- 标准错误
- 退出码
- 命令中使用的固定文件路径

## 判定依据

- 类型：标准输出判定 + 退出状态判定
- 成功条件：
  - `wego` 成功读取本地固定文件
  - 输出格式与所选前端匹配
  - 命令以 0 退出码结束
- 失败条件：
  - 无法读取固定文件
  - 所选前端的输出格式不正确
  - 运行仍然依赖实时网络访问

## 负面案例

- 将固定文件路径替换为不存在的文件，并确认判定机制将任务标记为失败。

## 边界情况

- 如果输出仍可正确解析，允许语义等价的 JSON 美化格式存在差异。
