# TUI 候选项目记录

## 作用

这个文件用于统一记录除 `wego` 之外、值得纳入 benchmark 池进一步测试的 TUI 项目。

它的目标不是替代每个项目自己的 benchmark 目录，而是作为候选池总表，帮助你后续做横向比较、筛选和立项决策。

## 使用方式

1. 每发现一个值得研究的 TUI，就新增一个条目。
2. 尽量按统一字段填写，方便后续比较。
3. 信息不完整时也可以先记下来，后面再补。
4. 真正入选 benchmark 的项目，后续再单独建立 `benchmarks/<project-name>/` 目录。

## 状态说明

- `待初筛`：刚加入列表，还没有深入检查。
- `调研中`：已经开始看 README、运行方式、依赖和交互形式。
- `候选保留`：整体合适，建议继续推进。
- `已选入 benchmark 池`：决定为它建立正式 benchmark 资产。
- `已淘汰`：看过后判断不适合当前研究目标。

## 建议字段

- `项目名`：项目名称。
- `仓库地址`：GitHub 或其他源码地址。
- `项目类型`：例如文件管理器、系统监控、终端 UI 应用、CLI 风格工具、编辑器等。
- `交互强度`：低 / 中 / 高。
- `外部依赖`：是否依赖网络、API key、特殊环境或本地服务。
- `安装与运行难度`：低 / 中 / 高。
- `适合评测的能力`：这个项目最适合拿来测什么。
- `不适合评测的能力`：这个项目不适合支撑哪些结论。
- `benchmark 适配度`：强 / 中 / 弱。
- `当前状态`：从上面的状态说明里选一个。
- `备注`：其他需要保留的观察。

## 候选条目

### 项目：wego

- 项目名：`wego`
- 仓库地址：当前工作区项目
- 项目类型：天气查询终端工具，偏 CLI / terminal output
- 交互强度：低
- 外部依赖：在线 backend 可能依赖网络或 API key；`json` backend 可本地离线运行
- 安装与运行难度：低
- 适合评测的能力：配置初始化、frontend 切换、缓存行为、离线 oracle 稳定性、agent 基础执行可靠性
- 不适合评测的能力：复杂焦点导航、多面板交互、全屏 TUI 可用性、重键盘探索行为
- benchmark 适配度：中
- 当前状态：已选入 benchmark 池
- 备注：适合做 pilot project，但不足以单独代表高交互 TUI

### 项目：lazygit

- 项目名：`lazygit`
- 仓库地址：https://github.com/jesseduffield/lazygit
- 项目类型：Git 仓库管理 TUI
- 交互强度：高
- 外部依赖：需要本地 Git 仓库；不依赖网络即可完成大量核心任务
- 安装与运行难度：低
- 适合评测的能力：多面板导航、焦点切换、状态理解、命令触发、局部确认操作、仓库上下文保持
- 不适合评测的能力：复杂文本编辑、大规模表单填写、依赖远程 API 的工作流
- benchmark 适配度：强
- 当前状态：已选入 benchmark 池
- 备注：本地可复现性较好，交互密度高，已建立 benchmark 资产

### 项目：nnn

- 项目名：`nnn`
- 仓库地址：https://github.com/jarun/nnn
- 项目类型：终端文件管理器
- 交互强度：高
- 外部依赖：主要依赖本地文件系统；可在隔离测试目录中运行
- 安装与运行难度：低
- 适合评测的能力：目录导航、文件选择、状态切换、批量操作前的上下文理解、快捷键驱动任务
- 不适合评测的能力：网络依赖型任务、复杂业务配置、富文本输出类 oracle
- benchmark 适配度：强
- 当前状态：候选保留
- 备注：环境要求轻，容易构造可重复 benchmark 任务，适合做文件系统操作类 TUI 基准

### 项目：btop

- 项目名：`btop`
- 仓库地址：https://github.com/aristocratos/btop
- 项目类型：系统监控 TUI
- 交互强度：中
- 外部依赖：依赖本机系统状态，不需要网络
- 安装与运行难度：低
- 适合评测的能力：信息定位、面板理解、低风险只读观察任务、系统状态浏览
- 不适合评测的能力：多步骤执行任务、持久性配置工作流、明确 side effect 验证
- benchmark 适配度：中
- 当前状态：已选入 benchmark 池
- 备注：已基于受控 Docker 负载构建 `datasets/btop/`，适合作为系统监控类 benchmark 的补充样本

### 项目：k9s

- 项目名：`k9s`
- 仓库地址：https://github.com/derailed/k9s
- 项目类型：Kubernetes 集群管理 TUI
- 交互强度：高
- 外部依赖：需要 Kubernetes 集群上下文；很多任务依赖真实集群状态
- 安装与运行难度：高
- 适合评测的能力：高密度导航、资源筛选、上下文切换、多层级状态理解、运维类真实任务
- 不适合评测的能力：低成本快速复现、完全离线 oracle、轻量本地基准
- benchmark 适配度：中
- 当前状态：调研中
- 备注：交互价值很高，但环境太重，建议放在第二阶段而不是立刻纳入第一批 benchmark

### 项目：Helix

- 项目名：`Helix`
- 仓库地址：https://github.com/helix-editor/helix
- 项目类型：终端文本编辑器
- 交互强度：高
- 外部依赖：主要依赖本地文件；部分高级能力可能依赖 language server
- 安装与运行难度：中
- 适合评测的能力：模态操作、复杂快捷键、编辑状态保持、屏幕区域理解、长链条交互任务
- 不适合评测的能力：低门槛快速 benchmark 落地、简单单步 oracle 验证
- benchmark 适配度：中
- 当前状态：待初筛
- 备注：研究价值高，但任务设计和 oracle 设计会明显更复杂，适合后续扩展而不是当前第一优先级
